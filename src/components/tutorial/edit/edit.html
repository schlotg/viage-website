Tutorial: Edit Component

Creating the Shopping List Edit Component

The only thing left to in order to have a fully functional app is to add the ability to edit an Item. The Edit functionality is very similar to the Add functionality. The only difference is that when the component initializes it needs to fetch that item's data and pre-populate the editable fields, save needs to save the element and not add it as a new one. It should be easy to take the existing add component and make it configurable enough to do re-use in the Edit state of our single page application.

Add an EDIT State

Much like we did for the ADD state, we will need to modify app.ts to have an EDIT state. Add an EDIT state to the States enum and add an entry in the router states so that your app.ts matches the following: 

${this.pp((this.code1), 'javascript')}

Modifying the Add Component HTML

When the Router instantiates a component for a state change, any parameters that go with the state change are passed in the component's init function as object. We will signal we are in edit mode by passing in an id that the component will use to use to fetch the item from the server. By using a template expression, the parser can evaluate whether params.id exists or not and make the page title reflect the correct state. Modify your shopping-list-add.html file to match the following: 

${this.pp(this.escapeHtml(this.code2), 'html')}

Modifying the Add Component TypeScript

The init function must to be modified to accept router params. The id portion of the parmeters must be passed into the setHTML() call so it can be evaluated when the HTML processes. Next, if an id is present, the item must be fetched from Service and then the input fields updated with the data. Finally, when Save is pressed, if in edit mode, then the Service just needs to save; otherwise it must add a new item. Update shopping-list-add.ts with the code below: 

${this.pp((this.code3), 'javascript')}

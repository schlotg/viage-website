Tutorial: Introduction

This tutorial walks through the building of a simple Shopping List App. In order to keep this tutorial simple and all about the Viage framework, the actual shopping list will be stored the browser's Local Storage instead of a server side database. It should not be difficult to adapt the service to make Restful API calls instead of storing and fetching from Local Storage. This tutorial assumes you have Nodejs installed and the latest version of the Viage CLI installed. If not, click on Getting Started and follow the instructions to install the Viage CLI. You can get out a working version of the final product at: Shopping List App and the github project for this tutorial is located at: Shopping List Project

What we are building

This web app will consist of a single page application that uses the Viage Router to provide 3 different views into the shopping list:

  1. Home - Provides a view of all the current list items
  2. Add - Provides a view to create an item to add
  3. Edit - Provides a view to edit an item in the list

Components, Routes, and Services

Viage is a minimal framework in which Apps are made from 3 basic building blocks:

  1. Components - Chunks of code and HTML that provide some sort of UI functionality
  2. Services - A class that provides services to components. This is a singleton and usually provides a mechanism to store and retrieve data.
  3. Routes - Configuration data that Viage router uses to change states. Router state changes involve rendering a component into a special portal HTML element

Command line Interface

Viage comes with a CLI tool that lets you easily scaffold out a new Viage project. It can also be used to create components an services within an existing project. This demo will be relying heavily on the Viage CLI to generate the project and for creation of Components and Services.

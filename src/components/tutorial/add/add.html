Creating the Shopping List Add Component

In order to see things in our list we need to be able to add them. This Component will also be a part of a router state called the ADD state. To get started we first need to create the ShoppingListAdd component. To create this component use the Viage CLI: 

${this.pp('viage component shopping-list-add', 'bsh')}

Copy and paste the following code into shopping-list-add.ts: 

${this.pp(this.escapeHtml(this.code1), 'javascript')}

Now create the file shopping-list-add.html in the same directory as the shopping-list-add.ts file and copy and paste the following code into it: 

${this.pp(this.escapeHtml(this.code2), 'html')}

How it Works

This component's HTML consists of a header element and a series of input elements for entering the Quantity, Description, Name, and some buttons to Save and go Back. The HTML is standard HTML with the exception of attach attributes added to the input elements and the buttons. This is simply a convienence mechahism so that the component has easy access to these elements.

Save

The init function adds the external HTML from the shopping-list-add.html file and then adds handlers to the Save and Back buttons. The handler for the Save button creates an Item object to send to the service so it can be added to the shopping list. It does this by getting the values from each of the input elements in this component's HTML which are stored in the components this.attachments member. Because the type of the attachments is HTMLElement, the item must be cast to a HTMLInputElement so that the TypeScript compiler won't complain when we try to get the value member. Yes the atttributes member could be typed as an 'any' object, but you should always try to maintain strict typing whenever possible so that the compilor can help catch any mistakes you might make.

Once the Item is constructed, shoppingListService.addItem(item) is called to add the item to the shooping list and save it. Then, the router is called to create the HOME state URL and it is instructed to go to that state. The back button handler just calls the router's back functionality which goes back one in the browser's history.

The ADD Route
